# 毕业设计整改任务清单

## 必须立即修改

1. **问题：云端后端不可用，无法完成登录和核心演示**
   - 修改目标：让部署链接能稳定完成登录、需求输入、模糊点确认、澄清问答、代码生成、一键测试、历史查看。
   - 建议做法：恢复 CloudBase Run 后端服务；逐个验证 `/api/login`、`/api/model_configs`、`/api/start`、`/api/detect`、`/api/answer`；用 `admin/admin123` 完成一次完整演示并截图。
   - 影响位置：部署 / PPT / 论文。

2. **问题：论文测试结论没有本地测试资产支撑**
   - 修改目标：让第6章测试内容可复现。
   - 建议做法：新增后端 pytest 用例、前端关键组件测试或说明、接口测试集合；保存 10 个对比实验任务的原始需求、澄清问答、生成代码、测试代码、运行日志。
   - 影响位置：代码 / 论文 / 查非项 / AI味。

3. **问题：论文实验数据前后冲突**
   - 修改目标：全文只保留一套实验数据和结论。
   - 建议做法：统一任务数量、对照组通过率、实验组通过率、绝对提升、相对提升；同步修改摘要、第6.5.2、第6.6、第7章和PPT第10页。
   - 影响位置：论文 / PPT / 查非项。

4. **问题：论文图表编号、章节结构存在明显错误**
   - 修改目标：目录、章节、图表编号全部连续、唯一、可交叉引用。
   - 建议做法：修正“八个章节”表述；处理目录中重复的 `2.2.3`；修正表6.14重复、图5.18和图5.19重复；更新目录和交叉引用。
   - 影响位置：论文 / 查非项。

5. **问题：真实密钥和数据库密码出现在 `backend/.env`**
   - 修改目标：提交材料中不包含真实密钥。
   - 建议做法：立即轮换已暴露的 DeepSeek、豆包、通义等 Key；删除真实 `.env`，改为 `.env.example`；README说明本地如何配置。
   - 影响位置：代码 / 部署 / 查非项。

6. **问题：README和部署说明严重不足**
   - 修改目标：答辩检查人员能按文档运行和验证系统。
   - 建议做法：新增根目录 `README.md`，包含项目简介、技术栈、目录结构、环境要求、数据库建库、后端启动、前端启动、部署地址、默认账号、常见问题；补 `deployment.md` 和 `test_report.md`。
   - 影响位置：代码 / 论文 / 部署。

7. **问题：PPT包含无证据性能指标和模板链接**
   - 修改目标：PPT只展示能被代码、论文、部署证明的成果。
   - 建议做法：删除模板网站链接；删除或降级“<2秒、>50用户、<512MB、真实业务场景实测”等无日志支撑指标；换成真实系统截图和演示流程。
   - 影响位置：PPT / 部署 / AI味。

## 建议尽快修改

1. **问题：论文摘要、绪论、总结口语化明显**
   - 修改目标：符合学位论文正式语体。
   - 建议做法：替换“发展飞快、成了、不过、关键得、路子、说白了、塞进”等表达；摘要中不要使用口语句式；英文摘要将 `This paper` 改为更合适表述。
   - 影响位置：论文 / 查非项 / AI味。

2. **问题：第1、2章概念铺陈偏多，与本人系统联系偏弱**
   - 修改目标：减少空泛背景，强化与系统实现的对应关系。
   - 建议做法：每个相关技术段落后补一句“本系统如何采用/取舍”，明确对应到 `Orchestrator`、`AnalyzerAgent`、`QuestionAgent`、`CodeAgent`、一键测试等。
   - 影响位置：论文 / AI味。

3. **问题：冲突检测表述与代码实现不完全一致**
   - 修改目标：论文/PPT表述与实际代码一致。
   - 建议做法：如果不新增冲突判定规则，就把“自动识别逻辑冲突”改为“支持状态快照回滚和重新澄清”；如果保留冲突检测，则补实现、测试用例和截图。
   - 影响位置：代码 / 论文 / PPT。

4. **问题：第6章测试表过于整齐，缺少真实过程**
   - 修改目标：让测试章节更像真实工程记录。
   - 建议做法：每类测试至少补一张运行命令/日志截图；功能测试表中加入失败或修复记录；说明测试环境、模型版本、API参数、运行日期。
   - 影响位置：论文 / 查非项 / AI味。

5. **问题：参考文献格式不统一**
   - 修改目标：符合学校参考文献规范。
   - 建议做法：按 GB/T 7714 逐条整理作者、题名、类型、来源、年份、卷期页码、URL和访问日期；重点检查发布日期晚于访问日期的条目。
   - 影响位置：论文 / 查非项。

6. **问题：部署构建与本地源码疑似不一致**
   - 修改目标：本地代码、GitHub代码、线上部署三者一致。
   - 建议做法：把部署所需 `basename`、后端 baseURL 配置纳入源码或环境变量；记录构建命令；重新构建并部署。
   - 影响位置：代码 / 部署 / PPT。

7. **问题：`hllm_memory.json` 含多类无关历史代码**
   - 修改目标：归档材料干净、与课题一致。
   - 建议做法：保留与智能代码生成演示有关的少量样例；删除明显无关的大段游戏、图书管理、记事本代码，或在文档中说明该文件只是示例经验库。
   - 影响位置：代码 / 论文。

8. **问题：数据库初始化不清楚**
   - 修改目标：新环境能稳定创建数据库和默认账号。
   - 建议做法：补建库SQL或脚本；说明 MySQL 版本、数据库名、字符集、默认管理员创建逻辑；不要依赖写死的本机密码。
   - 影响位置：代码 / 部署 / README。

## 可选优化

1. **问题：历史经验检索只用关键词重叠**
   - 修改目标：提升系统特色和论文说服力。
   - 建议做法：可选加入向量检索或更明确的相似度计算；若不实现，论文中不要把经验学习写得过强。
   - 影响位置：代码 / 论文 / PPT。

2. **问题：测试执行安全隔离较轻量**
   - 修改目标：提高自动测试模块可信度。
   - 建议做法：补充更严格的危险调用过滤、超时和资源限制说明；明确仅支持纯函数代码，不支持 Web/GUI。
   - 影响位置：代码 / 论文。

3. **问题：前端管理员判断只看用户名**
   - 修改目标：权限逻辑更规范。
   - 建议做法：登录后保存 `is_admin` 字段，前端路由守卫用 `is_admin` 判断；后端仍保持权限校验。
   - 影响位置：代码。

4. **问题：缺少持续集成或自动检查**
   - 修改目标：归档时更像完整工程。
   - 建议做法：增加简单 CI 或本地脚本，执行后端语法检查、pytest、前端构建。
   - 影响位置：代码 / README。

5. **问题：PPT答辩故事线偏概念化**
   - 修改目标：答辩时突出“做了什么、怎么做的、效果如何”。
   - 建议做法：减少背景页，把系统演示和实验结果放前；每个核心功能配一张真实截图和一句实现依据。
   - 影响位置：PPT。
